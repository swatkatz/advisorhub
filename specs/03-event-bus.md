# Spec: Event Bus

## Bounded Context

Owns: EventEnvelope, EventSource enum, EntityType typed constants, in-memory pub/sub bus. Provides the `Publish` and `Subscribe` interface used by all producers and consumers.

Does not own: Event payload interpretation (alert-generator's job), condition_key construction (alert-generator's job), event-triggering logic (each producer context decides when to emit), persistence to an external message broker (Kafka/NATS — out of scope for prototype). Does not own any database tables.

Depends on: Nothing — this is infrastructure. All other event-producing and event-consuming contexts depend on it.

Produces:
- `EventBus` interface: `Publish`, `Subscribe`
- `EventEnvelope` struct: the canonical event wrapper used across all contexts
- `EventSource` enum: `REACTIVE`, `TEMPORAL`, `ANALYTICAL`, `SYSTEM`
- `EntityType` typed constants: `Client`, `Account`, `Transfer`, `RESPBeneficiary` (extensible by other contexts)

## Contracts

### Input

No events consumed — this **is** the event infrastructure.

Data enters via `Publish`:
- Any context that produces events calls `bus.Publish(ctx, envelope)` with a fully-formed `EventEnvelope`
- The bus does not validate payload contents — it routes only

### Output

Interface exposed to all contexts:

```go
type EventBus interface {
    // Publish sends an event to all subscribers of envelope.Type.
    // Non-blocking: uses buffered channels so publishers are not blocked by slow consumers.
    Publish(ctx context.Context, envelope EventEnvelope) error

    // Subscribe returns a read-only channel that receives all events matching eventType.
    // Multiple subscribers per event type are supported.
    Subscribe(eventType string) <-chan EventEnvelope
}
```

### Data Model

**EventEnvelope** (struct, not a database table)

| Field | Type | Constraints |
|-------|------|-------------|
| ID | string | required, generated by publisher (UUID) |
| Type | string | required — event type name, e.g. `"OverContributionDetected"`, `"TransferStuck"` |
| EntityID | string | required — ID of the domain entity this event concerns |
| EntityType | EntityType | required — typed constant, e.g. `EntityTypeClient` |
| Payload | json.RawMessage | required — event-specific data, interpreted by consumers |
| Source | EventSource | required |
| Timestamp | time.Time | required, UTC |

**EventSource** (enum)

Values: `REACTIVE`, `TEMPORAL`, `ANALYTICAL`, `SYSTEM`

| Value | Used by |
|-------|---------|
| REACTIVE | Contribution Engine, Transfer Monitor, Seed Data Loader |
| TEMPORAL | Temporal Scanner |
| ANALYTICAL | Analytical Engine (mocked) |
| SYSTEM | Alert Lifecycle (AlertCreated, AlertUpdated, AlertClosed) |

**EntityType** (typed string constants)

```go
type EntityType string

const (
    EntityTypeClient          EntityType = "Client"
    EntityTypeAccount         EntityType = "Account"
    EntityTypeTransfer        EntityType = "Transfer"
    EntityTypeRESPBeneficiary EntityType = "RESPBeneficiary"
)
```

Extensible: other contexts may define additional `EntityType` values if needed. The bus does not validate against this set.

**No database tables.** All state is in-memory. Lost on process restart.

## State Machine

N/A — the event bus is stateless infrastructure. It routes events in real-time and does not track or transition state.

## Behaviors (EARS syntax)

**Publishing:**
- When `Publish(ctx, envelope)` is called, the system shall deliver the envelope to all channels subscribed to `envelope.Type`.
- When `Publish(ctx, envelope)` is called and there are no subscribers for that event type, the system shall complete without error (fire-and-forget).
- The system shall deliver events to subscribers non-blocking via buffered channels so that a slow consumer does not block the publisher or other subscribers.

**Subscribing:**
- When `Subscribe(eventType)` is called, the system shall return a read-only channel that receives only events matching that type.
- When `Subscribe(eventType)` is called multiple times with the same event type, the system shall return independent channels that each receive a copy of every matching event.

**Ordering:**
- The system shall deliver events to a given subscriber in the order they were published.

**Envelope:**
- Where `EntityType` is not one of the predefined constants, the system shall still accept and route the event (the bus does not validate EntityType values).

## Test Anchors

1. Given a subscriber to `"OverContributionDetected"`, when an event with `Type = "OverContributionDetected"` is published, then the subscriber receives the event on its channel.
2. Given a subscriber to `"OverContributionDetected"`, when an event with `Type = "TransferStuck"` is published, then the subscriber does not receive the event.
3. Given two subscribers to the same event type, when an event of that type is published, then both subscribers receive independent copies of the event.
4. Given no subscribers for a given event type, when an event of that type is published, then `Publish` completes without error.
5. Given a subscriber, when three events are published in order A, B, C, then the subscriber receives them in order A, B, C.
6. Given a slow consumer (not reading from its channel), when an event is published, then the publisher is not blocked (buffered channel absorbs the event).
7. Given an envelope with `EntityType = "SomeNewType"` (not a predefined constant), when published, then the event is routed normally to matching subscribers.
8. Given a subscriber registered after a previous event was published, when a new event is published, then the subscriber receives only the new event (no replay of historical events).
